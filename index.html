# 컴포넌트(Component, 줄여서 Comp) - 어떠한 속성들을 입력 받아 그에 맞는 React Element를 리턴해주는 역할을 함 - 자바스크립트 객체 형태로 존재하여 화면에 보이는 것을 기술 - 리엑트는 모든 페이지가 컴포넌트로 구성됨 - 하나의 컴포넌트는 여러 컴포넌트의 조합으로 구성될 수 있음 - 컴포넌트는 HTML 엘리먼트를 리턴하는 함수와 같음 - UI 조각인 컴포넌트를 모으면 전체 퍼즐 그림인 Web 페이지를 만들 수 있음 - 자바스크립트 함수처럼 작동하여 리엑트 엘리먼트를 반환 - 컴포넌트의 이름은 항상 대문자로 시작해야 함 (폴더는 소문자로 시작하는 카멜케이스로 작성, 컴포넌트를 만드는 파일은 대문자로 시작하는 카멜케이스로 작성) - 클래스형 컴포넌트 (Class Component)와 함수형 컴포넌트 (Function Component)로 나누어짐 # 프로퍼티(Property, 줄여서 Prop) - Component가 Element의 틀이라고 하면 그 안에 들어가는 재료가 Property (즉, 같은 Component라도 Property에 따라 내용이 달라진다.) - 컴포넌트의 속성 - 상위 컴포넌트가 하위 컴포넌트에 값을 전달할 때 사용 (부모에서 자식으로, 단방향 데이터 흐름을 가진다) (Parent ->[props]-> Child) (Chlid ->[event]-> Parent) - React 컴포넌트는 props를 이용하여 서로 통신함 # 클래스형 컴포넌트 (Class Component) - 함수형 컴포넌트에 비해 몇 가지 추가 기능이 있음 - 자바스크립트의 class를 이용하여 만들어진 형태의 컴포넌트 - vscode에서 rcc+Tab을 하면 기본 구조 생성 (React Class Component) # 함수형 컴포넌트 (Function Component) - pure 함수 같은 역할 (즉, 리엑트의 컴포넌트를 일종의 함수로 생각한다.) - 속성(props)이 없는 경우도 있음. (function Hello(){...) - 코드가 간단한 편 - vscode에서 rsf+Tab을 하면 기본 구조 생성 (React Statelsss Function) #기타 - 이미지는 import할 때 보통 public폴더 아니면 Asset 폴더에 저장 - 프로퍼티의 수는 여러가지가 될 수 있음 (하나의 props에 대해 props.name, props.color...) - 속성에는 {{}}를, 문자열 이외에는 {}를 사용 예) 부모 App.js와 함수형 컴포넌트 Main.js가 있다면 App.js에서
Main.js에서

{props.name}

- 프로퍼티는 자료형을 미리 지정할 수 있음 (defaultProps) (리엑트 엔진이 프로퍼티로 전달하는 값을, 효율적으로 알 수 있다.) 예) Main.js에서 import PropTypes from 'prop-types' [중략] Main.propTypes = { name: PropTypes.string } - 프로퍼티를 디폴트 설정하지 않는 경우, 그 프로퍼티를 필수 프로퍼티로 설정할 수 있음 (isRequired) 예) Main.js에서 import PropTypes from 'prop-types' [중략] Main.propTypes = { name: PropTypes.string.isRequired, } - 불리언(true, false)의 경우, 중괄호 없이 프로퍼티의 이름만 선언하면 됨 - 하나의 부모 컴포넌트에 여러 개의 자식을 쓸 수 있음
